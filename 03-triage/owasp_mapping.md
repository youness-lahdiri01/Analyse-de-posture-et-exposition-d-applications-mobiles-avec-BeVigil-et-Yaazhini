\# Mapping OWASP Mobile Top 10



\## M2 - Insecure Data Storage

\- Stockage de données sensibles (SharedPreferences)

\- Données non chiffrées localement



\## M3 - Insecure Communication

\- Utilisation de HTTP non sécurisé

\- Risque d’interception (MITM)



\## M5 - Insufficient Cryptography

\- Absence de chiffrement des données sensibles



\## M6 - Insecure Authorization

\- Activités Android exportées

\- Accès possible sans contrôle



\## M9 - Reverse Engineering

\- Application facilement analysable (Yaazhini)

\- Présence possible de secrets dans le code



\---



\## Conclusion



L’application InsecureBankv2 présente plusieurs faiblesses critiques alignées avec OWASP Mobile Top 10, notamment :



\- mauvaise gestion des données sensibles

\- communication non sécurisée

\- configuration Android faible



Ces vulnérabilités augmentent le risque de fuite de données et d’accès non autorisé.
